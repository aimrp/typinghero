# FRONTEND_GUIDELINES - 前端设计与 UI 规范

本文件定义打字英雄的前端设计和交互规范，目的是：

- 让 UI 风格稳定统一，而不是每次改动都“换一套风格”；
- 给 AI / 开发者明确的约束，避免随意调整颜色、间距和布局；
- 支撑后续继续美化时有章可循。

> 说明：以下内容基于当前 keyboard.html 的实现抽取而来，后续你可以按需要在这里继续收紧或细化规范。

## 1. 基础视觉风格

### 1.1 字体

- 首选：`"Microsoft YaHei", system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif`
- 适用范围：
  - 所有中文文案；
  - 按钮、输入框、仪表盘等 UI 文本。

### 1.2 颜色（建议调色板）

> 这里给出“概念层级”的颜色，实际十六进制值可以后续在此文件中统一收口。

- 主题色（Primary）：偏粉/玫红，用于 Logo 渐变、按钮高亮。
- 背景色（Background）：深色偏紫的渐变背景，营造“星光 / 夜空”的感觉。
- 强调色（Accent）：
  - 橙黄 / 金色：用于高连击、奖杯、终极胜利等提示；
  - 绿色：用于正确输入的文字高亮；
  - 红色：用于错误状态和警告。
- 文本颜色：
  - 主文本：白色或近白色；
  - 次要文本：半透明白（例如 rgba(255,255,255,0.7)）。

### 1.3 布局与间距

- 采用纵向分区布局：
  - 顶部：Logo + 连击容器 + 控制按钮；
  - 中部：文本展示区 + 输入框；
  - 底部：虚拟键盘 + 粒子特效画布。
- 间距规则（建议）：
  - 区块之间采用 16 / 24 / 32px 这几个固定步长；
  - 文本与容器边缘保持至少 12px 内边距；
  - 按钮之间保持 8 ~ 16px 的间距。

## 2. 组件级规范

### 2.1 Logo 与标题区

- 包含 SVG Logo（打字英雄字样），带简单发光 / 旋转动画。
- Logo 下方可放一句短标语，风格偏“热血 + 鼓励”。
- 不建议在此区域堆叠过多功能按钮，主要负责品牌感和氛围。

### 2.2 仪表盘（WPM / 准确率 / 进度）

- 每个指标使用圆形进度条 + 数字 + 文本标签组合。
- 颜色约定：
  - WPM：偏绿色，象征速度；
  - 准确率：偏蓝色或青色；
  - 进度：可用主题色或中性色。
- 动画：
  - 数值变化时，圆形进度条进行平滑过渡；
  - 文本变化时可使用轻微缩放或淡入效果。

### 2.3 文本展示区

- 显示当前练习的诗句：
  - 默认状态：高对比度的白色文字；
  - 已输入正确部分：可以使用高亮色（如绿色）或下划线强调；
  - 未输入部分：保持正常颜色。
- 来源标签（例如「来源：小学语文」）放在上方或左上角，字体较小。

### 2.4 输入框

- 默认状态：
  - 边框圆角；
  - 较高的高度，适应中文输入法候选窗的位置；
  - 背景略深以与总体背景区分。
- 正确输入状态：
  - 边框和内阴影偏绿色；
  - 可以配合轻微的发光。
- 错误状态：
  - 边框和阴影偏红色；
  - 可配合短暂的左右晃动（震动）动画。

### 2.5 虚拟键盘

- 布局：
  - QWERTY 键位，对应真实键盘；
  - 包含 Tab / Caps / Shift / Ctrl / Alt / Space / Enter / Backspace 等主要键。
- 样式：
  - 键帽为浅色或半透明亮色；
  - 边框略深，增加立体感；
  - 键帽上文字居中，使用与正文一致的字体。
- 交互：
  - 按下（真实键盘事件）时：
    - 键帽添加 `active` 样式（下压效果）；
    - 短暂高亮，约 120ms 后恢复；
    - 在键帽位置触发火花粒子。

## 3. 动效与特效规范

### 3.1 动画原则

- 动画服务于反馈和氛围，不应妨碍阅读和输入。
- 尽量使用短促、响应快的动画：
  - 按键按下：100~150ms；
  - 输入框错误震动：不到 300ms；
  - 统计数字变化：200~300ms 平滑过渡。

### 3.2 粒子与烟花

- 按键火花：
  - 在按键中心稍上方生成；
  - 向随机方向飞溅并快速消失；
  - 颜色可与 Combo 阶段挂钩。
- 底部火焰：
  - 随 Combo 提升，粒子数量与速度增加；
  - 高连击时可以增加蓝色/紫色火焰以营造“燃烧感”。
- 全屏烟花：
  - 在单句完成和终极胜利时触发；
  - 数量可随情境不同（普通完成 vs 通关）而变化。

## 4. 响应式与分辨率

当前主要使用场景是桌面端（Windows/macOS），但仍有一些基本约定：

- 最小可用窗口宽度建议不低于 1024px；
- 虚拟键盘和文本区在窗口变窄时优先保持横向不溢出：
  - 可以适当缩小字体和键帽尺寸；
  - 但不建议折行或多行键盘布局。

未来如果需要支持平板或更小分辨率，可以在本文件中新增断点设计说明。

## 5. 交互与文案风格

- 整体语气：
  - 「热血 + 轻度毒舌 + 鼓励」；
  - 嘲讽类文案要好玩但不恶意，适合中小学生。
- 文案类别：
  - 开局鼓励文案；
  - 慢速嘲讽文案（10s/20s/30s）；
  - 通关夸奖文案；
  - 游戏结束安慰文案。

后续如果增加多语言或多种语气（如“纯暖心模式”），可以在这里约定不同模式下的文案风格与选择规则。
